Figma API Wireframe Feasibility

Filed: February 27, 2026  ·  Requested by: K2S0 (for Cricket wireframe pipeline)  ·  Confidence: High

TL;DR

❌ Verdict: Not Possible via REST API

The Figma REST API is read-only for design content. It cannot create files, frames, rectangles, text nodes, or any design primitives. This is a deliberate architectural choice by Figma, not a gap waiting to be filled. The Plugin API can write to files, but requires the Figma desktop app open with user interaction — not viable for a headless agent.

1. What the Figma REST API Can (and Cannot) Do

Operation REST API Plugin API
Read file content / nodes YES YES
Export node images (PNG/SVG/PDF) YES YES
Read/write comments YES (both) NO
Manage webhooks, projects, teams YES NO
Create design nodes (frames, shapes, text) NO YES
Create new files from scratch NO PARTIAL
Work headlessly (no UI, no user) YES NO

The REST API exposes GET /v1/files/:key, image export endpoints, comment CRUD, webhooks, and library analytics. It has no POST or PUT endpoints for file content. Figma's own docs confirm: "REST API: Can only read the data from the file, can't write to the file."

A Figma Forum thread from a developer who tried this:
"I started using the figma API for the first time today and I was kind of surprised to find there are no endpoints for creating elements, be it documents, frames or shapes. Are there plans to add creation endpoints to the API?"
The official response: "You can use the Plugins API to edit files programmatically. There is no info on anything like this becoming available in the near future for the REST API."

2. Why the Plugin API Doesn't Help Us

The Plugin API can create frames and nodes:

// Plugin API only — requires Figma app open, user interaction
const frame = figma.createFrame()
frame.x = 50
frame.y = 50
frame.resize(375, 812)

const rect = figma.createRectangle()
rect.fills = [{ type: 'SOLID', color: { r: 0.9, g: 0.9, b: 0.9 } }]
frame.appendChild(rect)

The problem: it requires a user to have the Figma app open, be logged in, and run the plugin. Attempts to automate this via Selenium or headless browsers hit CAPTCHAs and login lockouts. The TalkToFigma MCP project (github.com/grab/cursor-talk-to-figma-mcp) works via a WebSocket bridge — but still requires Figma running with the plugin installed. Not headless.

3. Available Libraries (All Read-Only)

  • npm figma-api — 62.5K weekly downloads, TypeScript wrapper for REST API. Read-only for file content.
  • npm @neodx/figma — Typed API + asset export. Read-only.
  • npm figma-tools — fetchImages, watchFile, diffFiles. Read-only.
  • PyPI mcp-figma-tools — MCP server wrapping the REST API for file data, screens, node details. Read-only.

All libraries are wrappers around the same read-only REST API. None can create design content. The Figma MCP Server (Figma's own, beta as of Sept 2025) is also explicitly for design-to-code READ workflows, not creation.

4. Alternatives with Better APIs

🟢 Penpot (Best Option)

Open-source, self-hostable Figma alternative. Its Plugin API supports write operations. Key advantage: Penpot files are ZIP archives of SVG files — a well-documented, open format. An AI agent could potentially generate a valid .penpot file directly without any app interaction. Penpot is actively building MCP server support (Smashing Magazine, Jan 2026) for AI-powered design workflows. No proprietary binary formats, no gating.

🟡 Sketch (Mac-only, limited)

Has a JavaScript plugin API for write operations, but requires macOS and Sketch running. Not suitable for headless CLI use.

🟡 SVG → Figma Import

Generate SVG wireframes (Cricket can do this alongside HTML), then manually import into Figma. Loses Figma layer structure but gets you a design file. Still requires a human to open Figma and import. Not fully automated.

5. Practical Recommendation for Headless Agent Use

✅ Best Path Right Now

Keep generating HTML wireframes (Cricket's current output). HTML is actually a superior deliverable: shareable via URL, renderable in any browser, version-controllable, and requires zero proprietary tooling. If Figma files are required for stakeholder review, a human can import SVG into Figma — but that's optional, not critical.

If a proper "design file" format is a hard requirement in the future:

  • Penpot is the only viable path for fully headless programmatic design creation. Self-host Penpot, use its Plugin API, or generate .penpot files directly (ZIP of SVGs).
  • Monitor Figma's REST API changelog — they're investing in the platform (MCP server beta, granular OAuth scopes) but there's no announced timeline for write endpoints.
  • The TalkToFigma MCP bridge is a workaround but requires Figma open — not truly headless.
