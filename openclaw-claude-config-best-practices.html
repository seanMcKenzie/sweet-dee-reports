OpenClaw + Claude: Model Selection Best Practices

A practical guide for multi-agent teams — which model, when, and how to configure it.

TL;DR: As of early 2026, Sean's setup defaults every agent to claude-sonnet-4-6. That's not bad — Sonnet 4.6 is excellent — but it's leaving money on the table for light tasks (use Haiku) and performance on the table for heavy coding (use Opus). This report covers the full model lineup, per-agent config syntax, and concrete recommendations for each role in the K2S0 team.

1. Current Claude Lineup (Early 2026)

Haiku 4.5
$1 input / $5 output per MTok
Fastest. Near-frontier for simple tasks. Ideal for high-volume, low-complexity work and worker agents.
Sonnet 4.5 / 4.6
$3 input / $15 output per MTok
The daily driver. Balanced cost + capability. Sonnet 4.6 adds near-Opus coding performance. Sean's current default.
Opus 4.6
$5 input / $25 output per MTok
Flagship. 1M token context window. Best for complex coding, multi-step agents, deep reasoning. Feb 2026 release.

Key note: Opus 4.6 (Feb 2026) is now only 1.67× the price of Sonnet — far closer than the old 5× gap with Opus 4.1 ($15/$75). This changes the math significantly for heavy tasks. Haiku is still 3× cheaper than Sonnet, making it the right pick for coordinator routing, simple Q&A, and worker agents.

Legacy models (Opus 4.1, Claude 3.x) remain available but are not recommended for new setups. Haiku 3 is a budget option at $0.25/$1.25 but is significantly behind Haiku 4.5 on quality.

2. When to Use Each Model

Scenario Model Why
Complex coding / architecture Opus 4.6 Plans more carefully, sustains agentic tasks longer, catches its own mistakes. Worth the extra cost.
Daily coding tasks, feature work Sonnet 4.6 Handles 90% of coding tasks well at 40% lower cost than Opus. Good default.
Research & long documents Opus 4.6 1M token context window. Reads entire codebases or lengthy documents without losing thread.
Coordinator / routing tasks Sonnet 4.6 Over-powered with Haiku for pure routing, but Sonnet handles edge cases better. Acceptable.
Simple Q&A / voice interface Haiku 4.5 Sub-second latency. Perfect for quick answers where depth isn't needed.
QA / test generation Sonnet 4.6 Analytical, structured. Sonnet hits the cost-quality sweet spot here.
Creative / design briefs Sonnet 4.6 Creative tasks benefit from nuanced tone but rarely need Opus-level reasoning.
High-volume worker agents Haiku 4.5 Orchestrate with Sonnet, execute with Haiku. Standard multi-agent pattern.

3. OpenClaw Config: How to Set Models Per Agent

OpenClaw uses JSON5 (comments + trailing commas OK) for ~/.openclaw/openclaw.json. Model assignment works at two levels:

Global Default

"agents": {
  "defaults": {
    "model": {
      "primary": "anthropic/claude-sonnet-4-6"
    }
  }
}

Per-Agent Override (Recommended)

"agents": {
  "list": [
    {
      "id": "developer",
      "model": {
        "primary": "anthropic/claude-opus-4-6",
        "fallbacks": ["anthropic/claude-sonnet-4-6"]
      }
    },
    {
      "id": "research",
      "model": {
        "primary": "anthropic/claude-opus-4-6"
      }
    },
    {
      "id": "designer",
      "model": {
        "primary": "anthropic/claude-sonnet-4-6"
      }
    }
  ]
}

Channel-Level Override

"channels": {
  "modelByChannel": {
    "discord": {
      "CHANNEL_ID_HERE": "anthropic/claude-haiku-4-5"
    }
  }
}
⚠️ Heads up: As of Feb 2026, per-agent temperature and maxTokens inside agents.list[].model are not yet supported — config validation will reject them (GitHub issue #14494, marked as "enhancement"). Temperature is currently a defaults-only or slash-command setting. Use /think commands in chat to toggle reasoning depth per session.

Thinking / Reasoning Depth

  • /think off or minimal — No chain-of-thought. Fastest, cheapest.
  • /reasoning — Toggle extended thinking (uses more tokens, better on complex tasks)
  • Config default: set thinking: "low" in agent SOUL/AGENTS files as a convention note — actual runtime control is via slash commands.

5. Cost vs Capability: When to Spend More

Spend on Opus when: The task involves architectural decisions, large codebase review, agentic chains that run for minutes, or documents exceeding 200K tokens. Opus 4.6 at $5/$25 is a much easier call than Opus 4.1 was at $15/$75.

Stick with Sonnet when: The task is a standard feature, a moderate document, a design brief, or anything a skilled generalist can handle in one pass. Sonnet 4.6 delivers near-Opus coding performance at 40% lower cost.

Drop to Haiku when: The task is repetitive, low-complexity, or latency-critical. High-volume worker agents in multi-agent pipelines should default to Haiku. For reference, Haiku is 3× cheaper than Sonnet on input and output tokens.

Practical tip: Prompt Caching

Enable prompt caching for agents with large, stable system prompts (SOUL.md, AGENTS.md, USER.md). Cached tokens cost ~10% of normal input tokens. For research and developer agents with long context, this alone can reduce costs 40-60%.

Batch Processing

For non-real-time tasks (report generation, bulk analysis), OpenClaw can route through the Anthropic batch API for 50% cost savings. Ask K2S0 to queue non-urgent research tasks as batch jobs.
