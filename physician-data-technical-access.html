Physician Data Sources: Technical Access Deep-Dive

Implementation-ready technical documentation for every major physician data source

Date: February 26, 2026 Prepared for: Sean McKenzie / K2S0 Version: 1.0
TL;DR: The free CMS sources are genuinely accessible with no authentication or minimal registration. The NPPES NPI Registry, Medicare Part D & Part B data, Open Payments, and the Provider Data Catalog are all fully open — no API keys required to start. DEA data is a meaningful gap: there's no free public bulk API, you need a paid service or to be a DEA registrant yourself. All major commercial sources (AMA Masterfile, Definitive Healthcare, Doceree, Doximity) are well above $1,000/month with no documented affordable tiers.

The canonical physician identifier database — your foundation data source
FREE

~8.5 million provider records. Every licensed US healthcare provider has an NPI. Two access methods: bulk download for full dataset, API for real-time lookups.

# Lookup by NPI number
https://npiregistry.cms.hhs.gov/api/?version=2.1&number=1234567890

# Search cardiologists in Texas
https://npiregistry.cms.hhs.gov/api/?version=2.1&taxonomy_description=Cardiology&state=TX&enumeration_type=NPI-1&limit=200

# Search by name (wildcard supported after 2 chars)
https://npiregistry.cms.hhs.gov/api/?version=2.1&first_name=John&last_name=Sm*&limit=200&pretty=on

# Paginate with skip
https://npiregistry.cms.hhs.gov/api/?version=2.1&city=Boston&limit=200&skip=200

Gotchas & Notes

  • API caps at 1,200 records per search — use bulk download for full dataset extraction
  • state alone is not valid — must combine with another parameter
  • use_first_name_alias=True by default — returns Robert for Bob, etc. Set to False for exact match
  • Output has nested JSON arrays for taxonomies, addresses, endpoints (not flat)
  • No documented rate limit, but aggressive polling may get throttled
  • API v1.0 and v2.0 are retired — only v2.1 works
  • R package available: npi on CRAN wraps the API cleanly

1.6 DEA Registrations

No free public API exists. Significant access barrier. Solutions below.
NOT FREELY AVAILABLE
🔒 Gap alert: Unlike CMS data, the DEA does not publish a free, open bulk download or public API for registrant data. This is a real blocker if DEA verification is required in your system.

What Actually Exists

OptionAccess RequirementWhat You Get
DEA Registrant Validation ToolsetMust be an active DEA registrant yourselfValidate another registrant's status + limited dataset download
Algorithmic number validationNone — free to implementFormat & checksum validation only. Does NOT confirm active registration.
DEA Lookup.com (commercial)Paid subscriptionFull registrant data, daily updates, REST API

Free Option: DEA Number Checksum Algorithm

DEA numbers follow a deterministic checksum. You can validate format without any API call:

Format: [2 letters][7 digits] Letter 1: Registrant type (B=hospital, C=practitioner, M=mid-level, X=suboxone...) Letter 2: First letter of registrant's last name Checksum validation: A = digit1 + digit3 + digit5 B = (digit2 + digit4 + digit6) × 2 digit7 must equal last digit of (A + B) Example: AB1234563 A = 1+3+5 = 9 B = (2+4+6) × 2 = 24 A+B = 33 → last digit = 3 ✓ (matches digit7)

2.1 AMA Physician Professional Data (Masterfile)

1.4M physician records from medical school through retirement. Most complete physician dataset.
$30K–$150K+/year
Pricing
$30,000–$150,000+/year. No affordable tier.
Access Method
Bulk flat file delivery via SFTP
Format
CSV / fixed-width flat file (AMA spec)
Update Freq
Monthly refresh files
Record Count
~1.4M physicians (MDs, DOs, residents, students)
Authentication
License agreement + SFTP credentials

Key differentiator vs. CMS sources: AMA covers physicians who do NOT participate in Medicare, including residents, international medical graduates, and early-career physicians. 2,200+ data elements vs. ~330 in NPPES.

How to license: Contact AMA directly (ama-assn.org → Physician Data Solutions) or through licensed resellers: Redi-Data, IQVIA/IMS. Requires data use agreement specifying intended use.

No REST API for full dataset queries. Bulk file delivery only. Monthly update files delivered incrementally.

Lower-cost alternative: Redi-Data (redidata.com) sells targeted subsets — you buy a physician list by specialty/geography rather than the full file. Better for targeted use cases; more affordable than full licensing.

~$50K+/year

API Base URL

Token Endpoint
POST /v4/token
Protocol
REST / OData

JSON (API), CSV/XML (reports)

None official; standard HTTP client

Authentication Flow

# Step 1: Get Bearer token
curl -X POST \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password&username={user}&password={pass}" \
  "https://api.defhc.com/v4/token"
# Returns: { "access_token": "DQ8udZDz...", "token_type": "Bearer" }

# Step 2: Use token in all requests
curl -X GET \
  -H "Authorization: Bearer {access_token}" \
  "https://api.defhc.com/v4/odata-v4/Hospitals"

Key Endpoints

EntityEndpoint
HospitalsGET /v4/odata-v4/Hospitals
Hospital by IDGET /v4/odata-v4/Hospitals({id})
Hospital + ExecutivesGET /v4/odata-v4/Hospitals?$expand=Executives
ExecutivesGET /v4/odata-v4/Executives
Reports (list)GET /v4/Reports?returnType=csv
Custom Report downloadGET /v4/Reports/{reportName}

Important Gotcha

⚠️ Physician data access depends on your specific license. More importantly: the API works primarily through custom reports configured by your Definitive account manager. You don't freely query arbitrary physician fields — you define report templates with your account team, then pull them via the Reports endpoint. Less flexible than it appears.

Programmatic HCP advertising network — 2,000+ medical platforms, 6M+ verified HCPs
MEDIA PLATFORM
🚨 Not a data licensing product. Doceree is a media/advertising platform for reaching physicians with ads. It uses NPI-level targeting internally but does not sell or expose physician data. If Sean needs to advertise to physicians this is relevant; if he needs to identify or export physician data, it is not.
